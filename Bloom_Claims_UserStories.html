Must Have Stories
EPIC 1 Application Foundation & Authentication Phase 1 17 pts
Story ID User Story Acceptance Criteria Priority Pts
EP1-US001 As a claims examiner, I want to log in using my company credentials so that I can securely access the claims portal.
  • OAuth 2.0 SSO redirects to ServiceNow login
  • JWT token stored securely in session
  • Failed login shows a clear error message
  • Session expires after configurable inactivity period
 Must Have 5
EP1-US002 As a system admin, I want user roles automatically assigned from ServiceNow so that access control is consistent with enterprise permissions.
  • Role retrieved from ServiceNow profile on login
  • Examiner/supervisor/manager roles map to correct views
  • Role displayed in user menu
 Must Have 3
EP1-US003 As a demo presenter, I want to switch between user personas so that I can demonstrate different role experiences without multiple logins.
  • Persona switcher visible in demo mode only
  • Switching loads correct dashboard and permissions
  • Demo badge visible when active
 Must Have 2
EP1-US004 As any user, I want a consistent navigation sidebar so that I can quickly move between portal sections.
  • Sidebar shows only permitted nav items per role
  • Active section highlighted
  • Collapses to icon-only mode
  • Fully accessible by keyboard
 Must Have 3
EP1-US005 As any user, I want to toggle between light and dark mode so that I can work comfortably in different environments.
  • Toggle persists across sessions via localStorage
  • All MUI components respect theme tokens
  • Charts and badges adapt to theme
 Should Have 2
EP1-US006 As any user, I want to switch between L&A and P&C product lines so that I can view claims relevant to my current work context.
  • Product line switcher available in header
  • All data and labels update on switch
  • Defaults to L&A on login
 Should Have 2
EPIC 2 Dashboards & Claim Search Phase 1 26 pts
Story ID User Story Acceptance Criteria Priority Pts
EP2-US001 As a claims examiner, I want to see my assigned claims on my dashboard so that I can prioritize my work queue.
  • Only assigned claims shown
  • Claims sorted by SLA urgency by default
  • Claim count displayed in header
  • Dashboard loads within 3 seconds
 Must Have 5
EP2-US002 As a claims examiner, I want to see SLA status badges on each claim so that I immediately know which claims are at risk.
  • On-track (green), At-risk (amber), Overdue (red) badges
  • Days remaining shown on each badge
  • Tooltip explains the SLA rule applied
 Must Have 3
EP2-US003 As a claims examiner, I want to see FastTrack/STP eligibility on each claim so that I can prioritize straight-through processing.
  • FastTrack badge shown for eligible claims
  • STP badge for fully automated claims
  • Clicking badge shows eligibility criteria
 Must Have 2
EP2-US004 As any user, I want to search for claims by ID, insured name, policy number, status, or date range so that I can quickly locate specific claims.
  • Search returns results within 2 seconds
  • Supports partial match on name and ID
  • No-results state displayed clearly with guidance
 Must Have 3
EP2-US005 As a claims supervisor, I want to see all my team's claims in my dashboard so that I can manage workload distribution.
  • All team claims visible regardless of individual assignment
  • Filterable by examiner
  • Team metrics shown: total open, SLA at-risk count
 Must Have 5
EP2-US006 As a claims manager, I want a risk management dashboard with performance analytics so that I can make informed operational decisions.
  • Written premium YTD, approval/decline rates, SLA compliance %, FastTrack %
  • Date range filter applied to all metrics
  • Product line filter available
 Must Have 8
EPIC 3 Claims Workbench Phase 2 30 pts
Story ID User Story Acceptance Criteria Priority Pts
EP3-US001 As a claims examiner, I want to open a claim workbench with a full timeline so that I can see the complete history of a claim.
  • Timeline shows all events chronologically
  • Each event has timestamp, actor, and description
  • Timeline is scrollable and printable
 Must Have 5
EP3-US002 As a claims examiner, I want to view the Policy 360 tab so that I can see all policy details relevant to the claim.
  • Shows: policy number, type, status, effective/term dates, face amount, riders
  • Beneficiary list with allocations displayed
  • Data sourced from ServiceNow FSO
 Must Have 5
EP3-US003 As a claims examiner, I want to track requirements in the workbench so that I can manage what documents and information are still outstanding.
  • Requirements listed with status: Pending / Received / In Progress / Complete
  • Can update status inline; received date auto-stamped
  • IGO (In Good Order) flag visible per requirement
 Must Have 5
EP3-US004 As a claims examiner, I want to view and manage documents in the workbench so that all claim documents are accessible in one place.
  • Documents grouped by type in accordion sections
  • Each document shows name, type, and upload date
  • Viewer opens in-panel; upload button available
 Must Have 5
EP3-US005 As a claims examiner, I want to see a claim progress bar so that I understand how complete the claim processing is.
  • Progress % calculated from completed requirements and steps
  • Visible at top of workbench
  • Updates automatically on requirement completion
 Must Have 2
EP3-US006 As a claims examiner, I want to Approve, Deny, or Hold a claim so that I can take action on processed claims.
  • Action buttons in workbench header
  • Confirm dialog before any action is taken
  • Status updates in ServiceNow; audit trail entry created
  • Email notification triggered to claimant
 Must Have 5
EP3-US007 As a claims examiner, I want to add work notes to a claim so that I can document my investigation and communicate with teammates.
  • Work notes threaded by date with author name and timestamp
  • Rich text entry supported
  • Notes sync to ServiceNow on save
 Must Have 3
EPIC 4 FNOL & Intake Phase 2 21 pts
Story ID User Story Acceptance Criteria Priority Pts
EP4-US001 As an intake specialist, I want a 3-step FNOL wizard so that I can capture all required claim information in a guided, error-free process.
  • Step 1: claim type, policy #, insured name, date of death, description
  • Step 2: claimant name, email, phone, relationship
  • Step 3: document upload; cannot advance with validation errors
 Must Have 8
EP4-US002 As an intake specialist, I want to drag and drop documents during FNOL so that I can upload death certificates and IDs quickly.
  • Drag-and-drop zone accepts PDF, JPG, PNG up to 25 MB
  • Upload progress shown; uploaded files listed
  • Files can be removed before submission
 Must Have 3
EP4-US003 As an intake specialist, I want to receive a claim number after FNOL submission so that I can provide confirmation to the claimant.
  • Claim number displayed on success screen
  • Next-steps message shown
  • Option to print or email confirmation
 Must Have 2
EP4-US004 As a claims examiner, I want to use an FNOL Workspace so that I can manage all incoming claim notifications in one view.
  • Lists all new FNOL submissions with submission date, claimant name, claim type
  • Can open any submission for review
  • Filterable by status
 Should Have 5
EP4-US005 As an intake specialist, I want FNOL submission to trigger death verification so that accuracy is validated at intake.
  • LexisNexis verification triggered on submit
  • Verification status stored on claim; examiner notified of result
  • Stub mode available when credentials not yet configured
 Should Have 3

Story ID User Story Acceptance Criteria Priority Pts

  • Death case record fetched on workbench open
  • Populated into Death Event Panel automatically
  • Error state displayed if no record found
 Must Have 2
EP5-US003 As a claims examiner, I want to see the LexisNexis death verification status on the claim so that I can confirm the death is verified before proceeding.
  • Verification status badge: Verified / Unverified / Pending in claim header
  • Timestamp of last verification shown
  • Re-trigger verification button available
 Should Have 3
EPIC 6 Beneficiary Analysis & Party Management Phase 3 18 pts
Story ID User Story Acceptance Criteria Priority Pts
EP6-US001 As a claims examiner, I want to view all beneficiaries with their details so that I can assess who is entitled to the benefit.
  • Shows: name, masked SSN (reveal on click), relationship, allocation %, birth date, status
  • Data sourced from ServiceNow FSO
 Must Have 3
EP6-US002 As a claims examiner, I want to add or edit parties on a claim so that I can correct or update insured, claimant, and beneficiary information.
  • Party form: name, SSN, DOB, relationship, contact info
  • Validates all required fields before save
  • Saves to ServiceNow; audit trail entry created
 Must Have 5
EP6-US003 As a claims examiner, I want the system to automatically analyze beneficiaries for conflicts so that I can identify and resolve eligibility disputes early.
  • AI analysis auto-triggers on workbench open
  • Findings show conflict description, severity, and recommended resolution
  • Examiner can acknowledge and note action taken
 Must Have 8
EP6-US004 As a claims examiner, I want to see beneficiary conflict severity levels so that I can triage disputes appropriately.
  • HIGH conflicts shown as red alert; MEDIUM as amber; LOW as informational
  • Conflict counts shown in workbench header badge
 Must Have 2
EPIC 7 Intelligent Document Processing Phase 3 24 pts
Story ID User Story Acceptance Criteria Priority Pts
EP7-US001 As a claims examiner, I want uploaded documents to be automatically classified by type so that I don't have to manually tag every document.
  • Classification runs within 30 seconds of upload
  • Document type label applied with confidence score shown
  • Examiner can correct classification if needed
 Must Have 8
EP7-US002 As a claims examiner, I want key fields extracted from uploaded documents so that I don't have to manually re-key data that is already in the document.
  • Extracted fields shown alongside source document
  • Each field displays a confidence percentage
  • Extracted fields mapped to claim form sections
 Must Have 8
EP7-US003 As a claims examiner, I want extracted data to auto-populate claim fields so that data entry is minimized.
  • Auto-populated fields highlighted in form
  • Examiner reviews and confirms or edits before saving
  • Original extracted value preserved in audit
 Must Have 5
EP7-US004 As a claims examiner, I want to review and override auto-populated data so that I maintain control over what is saved to the claim record.
  • All auto-populated fields are editable
  • Override saves new value and logs original extraction
  • Confidence score visible throughout review
 Must Have 3
EPIC 8 AI Insights & Anomaly Detection Phase 3 23 pts
Story ID User Story Acceptance Criteria Priority Pts
EP8-US001 As a claims examiner, I want an AI Insights panel in the workbench so that I can quickly see if there are any risk indicators on the claim.
  • AI panel visible on workbench load
  • Shows overall risk level: HIGH / MEDIUM / LOW / CLEAR
  • Expandable to full modal; loads within 5 seconds
 Must Have 8
EP8-US002 As a claims examiner, I want each AI finding to include a description, evidence, and recommendation so that I understand why a risk was flagged.
  • Finding card shows: severity badge, title, evidence bullets, recommended action, PASS/FAIL indicator
  • Can be dismissed with an examiner note
 Must Have 5
EP8-US003 As a claims examiner, I want an AI-generated claim summary so that I can quickly understand the key facts and risks without reading the full file.
  • Summary paragraph auto-generated on workbench open
  • Includes: insured, claim type, amount, key dates, active risk count
  • Refreshes when claim data changes
 Must Have 5
EP8-US004 As a claims manager, I want anomaly trends visible in the manager dashboard so that I can see if fraud patterns are increasing.
  • Anomaly count by severity charted over time
  • Top anomaly types listed
  • Filterable by date range and examiner
 Nice to Have 5

EPIC 14 Non-Functional & Accessibility All Phases 15 pts
Story ID User Story Acceptance Criteria Priority Pts
EP14-US001 As any user, I want the portal to be keyboard navigable so that users with motor impairments can use all features.
  • All interactive elements reachable by Tab key
  • Focus indicators clearly visible
  • No keyboard traps; all modals closeable with Escape
 Must Have 5
EP14-US002 As any user, I want all PII fields (SSN, DOB, account numbers) masked by default so that sensitive data is not exposed on screen.
  • SSN shows as ***-**-XXXX; DOB shows as **/**/XXXX
  • Reveal icon toggles visibility per field
  • Reveal action logged in audit trail
 Must Have 3
EP14-US003 As any user, I want the portal to load in under 3 seconds on a standard enterprise network so that productivity is not impacted.
  • Lighthouse performance score ≥ 80
  • Dashboard TTI ≤ 3s on 10 Mbps connection
  • Lazy loading implemented on non-critical components
 Must Have 5
EP14-US004 As any user, I want the portal to work on Chrome, Edge, and Safari (latest 2 versions) so that all staff can use their preferred browser.
  • E2E tests pass on Chrome, Edge, and Safari
  • No layout breaks or browser-specific JavaScript errors
 Must Have 2
